# Heatwave School Closures Dashboard — Model Documentation

*Generated from `src/model.js` and `src/App.jsx`. All formulae reflect the exact code logic.*

---

## 1. Data inputs

### 1.1 Local authority data (`src/data/localAuthorities.json`)

Each local authority record exposes six fields used by the model:

| Field | Type | Source | Description |
|---|---|---|---|
| `schools` | integer | GIAS | Count of open mainstream/nursery establishments |
| `pupils` | integer | GIAS | Total pupil roll across all phases |
| `pupilsPrimary` | integer | GIAS | Pupils in Reception–Y6, Nursery, and Middle deemed primary |
| `pupilsSecondary` | integer | GIAS | Pupils in Y7–Y13, All-through, and Middle deemed secondary |
| `amber[levelKey]` | float | UKCRI | Median amber heat-health alert events per year at each Global Warming Level |

`amber` is keyed by GWL: `"0.61"` (recent climate), `"1.5"`, `"2"`, `"3"`, `"4"`.

The `pupilsPrimary` and `pupilsSecondary` fields are generated by `scripts/build_data.mjs` during the data build step. Run `npm run build-data` after modifying the build script or updating the GIAS extract.

### 1.2 Parameters

All parameters are user-adjustable via the Sources & assumptions panel. They fall into two categories:

**Scenario assumptions** (amber dot in UI — no external source):

| Key | Default | Range | Description |
|---|---|---|---|
| `amberToRedFraction` | 0.30 | 0–1 | Share of amber alerts that escalate to red severity |
| `schoolClosureFraction` | 0.35 | 0–1 | Share of schools that close during a red alert (the top-of-page control; ~35% ≈ the ~2,000 schools closed in the south/south-east during the June 2026 red alert) |
| `redAlertDurationDays` | 2 | 1–7 days | Duration of a closure event |
| `supervisionDiscountKS3` | 0.55 | 0–1 | Supervision factor for Y7–9 families (see §3) |

**Evidenced figures** (green dot in UI — sourced):

| Key | Default | Source |
|---|---|---|
| `costPerFamilyPerDay` | £145 | HM Treasury Green Book value of time; ONS ASHE 2024 median full-time earnings ≈ £146/day |
| `realizedDisruptionFraction` | 0.50 | IEA, "What are the real costs of the recent strike wave?" (see §2.3a) |
| `childrenPerFamily` | 1.75 | ONS Families and Households 2024 |
| `schoolDaysPerYear` | 190 | DfE statutory minimum |

## 2. Single-event economic impact

### 2.1 Schools and pupils affected

```
schoolsClosed  = round(la.schools × schoolClosureFraction)
pupilsAffected = la.pupils × schoolClosureFraction
```

### 2.2 Phase-split affected pupils

The model splits affected pupils into primary and secondary buckets before computing family disruption. This is what enables the supervision discount (§3) to apply differentially by phase.

```
primAffected = (la.pupilsPrimary  ?? la.pupils) × schoolClosureFraction
secAffected  = (la.pupilsSecondary ?? 0)        × schoolClosureFraction
```

The `??` fallback means that if phase data is absent (older data builds), all pupils are treated as primary — the conservative assumption, since primary attracts the highest supervision factor.

### 2.3 Effective family count

The core novelty of the economic model is the `effectiveFamilies` function, which applies two corrections simultaneously:

**Correction 1 — supervision discount by phase.** Not all families experience genuine work disruption on a closure day. The discount varies by key stage:

| Phase | Factor | Rationale |
|---|---|---|
| Primary (R–Y6) | 1.00 | Full supervision required |
| KS3 (Y7–9) | `supervisionDiscountKS3` (default 0.55) | Partial — most parents adjust but some do not |
| KS4 (Y10–11) | 0.20 (fixed) | Mostly independent; low disruption |
| KS5 (Y12–13) | 0.05 (fixed) | Effectively independent |

The secondary roll is assumed to split equally across seven year groups, giving 3/7 KS3, 2/7 KS4, 2/7 KS5 by pupil count:

```
primWeighted = primAffected × 1.00
ks3Weighted  = secAffected × (3/7) × supervisionDiscountKS3
ks4Weighted  = secAffected × (2/7) × 0.20
ks5Weighted  = secAffected × (2/7) × 0.05
```

**Correction 2 — multiple children per household.** A household with two children at the same school generates one disruption event, not two. Dividing by `childrenPerFamily` converts effective weighted pupils into effective households:

```
potentialFamilies = (primWeighted + ks3Weighted + ks4Weighted + ks5Weighted)
                     / childrenPerFamily
```

### 2.3a Realised disruption

Not every supervision-weighted family actually loses a paid working day — many arrange informal cover (a relative, a neighbour, a remote-work adjustment) even when a child's school is shut. `potentialFamilies` is therefore discounted by `realizedDisruptionFraction` (default 0.50) to give the final `familiesAffected` used for costing:

```
familiesAffected = potentialFamilies × realizedDisruptionFraction
```

This mirrors the IEA's own analysis of the 2022–23 UK strike wave ("What are the real costs of the recent strike wave?", iea.org.uk): an ONS-style survey found 59% of parents said a school closure would force them to work fewer hours or not at all, but the author judged it conservative to assume only half of that group actually did so once faced with a real closure. The dashboard applies the same 50% halving on top of the phase-based supervision discount above, rather than in place of it.

A sense-check against the IEA's own headline number (≈£180m per national school-strike day, 2023 prices) supports this: scaling the dashboard's model to a hypothetical full nationwide one-day closure at the current £145/day wage and *without* a realised-disruption discount would predict a cost roughly 2.4–3.5× the IEA's inflation-adjusted figure. Solving for the realisation rate implied by the IEA's own numbers gives ≈29% — close to the 50% figure used here (50% is the IEA author's own stated assumption; ≈29% is the rate implied if their headline figure is taken at face value). 50% was kept as the default since it is the rate the source explicitly states, rather than a back-solved figure.

### 2.4 Economic cost

```
economicImpact = familiesAffected × costPerFamilyPerDay × redAlertDurationDays
```

`costPerFamilyPerDay` is valued as the **opportunity cost of the caregiver's lost day** following HM Treasury Green Book guidance on the value of time (lost output at the market wage), anchored to ONS ASHE 2024 median full-time earnings (≈ £146/day). This supersedes the earlier childcare-replacement proxy, which valued only paid substitute care rather than lost economic output. It is applied only to the supervision-weighted family count, not to raw pupils — so the phase discount is already embedded before this multiplication.

## 3. Single-event learning impact

```
learningDaysLost = pupilsAffected × redAlertDurationDays
```

Learning loss is expressed as raw pupil-days of instruction lost. It uses the unweighted `pupilsAffected` count — **every affected pupil loses the same number of instruction days regardless of age or phase.** The supervision discount does not apply here; it is strictly an economic construct.

In the UI, this is also expressed as:

```
daysLostPerPupil = redAlertDurationDays           // exact, not modelled
yearFraction     = daysLostPerPupil / schoolDaysPerYear
```

No catch-up discount, learning multiplier, or long-run earnings conversion is applied. The model treats heat closures as brief isolated events where instructional time is simply lost.

---

## 4. Annual impact

Annual impact scales single-event impact by the expected number of red-alert events per year. The amber alert frequency at each GWL comes directly from the UKCRI data; the red event rate is derived by applying the escalation fraction:

```
amber            = la.amber[levelKey]            // median events/year from UKCRI
redEventsPerYear = amber × amberToRedFraction

annualEconomic = redEventsPerYear × economicImpact    // from single-event calc
annualLearning = redEventsPerYear × learningDaysLost  // from single-event calc
```

The single-event figures (`economicImpact`, `learningDaysLost`) are computed once via `perEventImpact()` and then scaled. Changing the GWL changes `amber` and therefore `redEventsPerYear`, but does not change the per-event impact figures — those depend only on school/pupil characteristics and the closure scenario parameters.

---

## 5. Decade totals

The dashboard's climate tab aggregates across a decade by simple multiplication:

```
decadeEconomic = annualEconomic × 10
decadeLearning = annualLearning × 10
eventsPerAuth  = (redEventsPerYear summed across LAs / number of LAs) × 10
```

Note that `eventsPerAuth` is an **average per authority**, not a sum across authorities. Summing would imply each authority experienced separate independent events; the average represents a single heatwave event affecting all authorities simultaneously, which is the correct interpretation for a national red-alert closure.

## 6. Career learning loss

```javascript
// careerLearningLoss(la, levelKey, p)
closureDaysPerYear = redEventsPerYear × redAlertDurationDays × schoolClosureFraction
careerYears        = 14   // Reception (age 4) through Year 13 (age 18)
careerLoss         = closureDaysPerYear × 14
```

This answers: *if the climate stabilises permanently at this GWL, how many closure days does a pupil accumulate across their entire school career?*

This is explicitly **not** a warming trajectory — it does not model the path from current climate to the target GWL. It is a snapshot: what the annual closure rate would be in a world that has reached and stabilised at a given GWL, applied across 14 school years.

In the UI this is averaged across selected LAs and also expressed as a fraction of a single school year:

```
careerYearFraction = careerLoss / schoolDaysPerYear
```

## 7. Aggregation across local authorities

All multi-LA totals use a simple reduce:

```javascript
// totals(las, fn)
las.reduce((acc, la) => {
  const r = fn(la)
  for (const k in r) acc[k] = (acc[k] || 0) + r[k]
  return acc
}, {})
```

Economic costs and pupil-day learning losses are summed. The `eventsPerAuth` figure divides the summed red events by the number of LAs before multiplying by 10 (see §5). The `careerLoss` figure shown in the UI is the mean across selected LAs, not the sum.

## 8. Warming levels

| Key | Label | Policy context |
|---|---|---|
| `"0.61"` | Recent | UKCRI baseline / current-climate calibration |
| `"1.5"` | 1.5°C | Paris Agreement lower target |
| `"2"` | 2°C | Paris Agreement upper target; CCC planning GWL |
| `"3"` | 3°C | Current-policy trajectory (intermediate) |
| `"4"` | 4°C | High-emissions upper bound; CCC stress-test |

The amber alert frequencies at each GWL come from the UKCRI file `heathealth_null_events_ghadgem_dt_lau1.csv` — median values from the UKCP18 Global HadGEM ensemble, LAU1 district level, rolled up to GIAS education LA boundaries by simple mean in the build script.

## 9. Key modelling assumptions and limitations

**Equal year-group size within secondary.** The 3/7–2/7–2/7 KS3/KS4/KS5 split assumes equal numbers of pupils per year group. In practice sixth forms are typically smaller than KS3/KS4 year groups, which means the model slightly understates the average supervision factor for secondary schools (and therefore slightly understates economic impact).

**All-through schools treated as secondary.** GIAS does not provide a primary/secondary pupil split within all-through schools. The build script assigns all-through pupils to `pupilsSecondary`. This is conservative — it understates economic impact for all-through schools with large primary-age populations.

**GWL snapshot, not trajectory.** `careerLearningLoss` applies a fixed GWL's annual closure rate across 14 years. A pupil starting school today will experience rising closure rates as warming increases, not a fixed rate. The career figure therefore understates cumulative impact for pupils whose career spans a warming period.

**Amber-to-red escalation is uniform.** The `amberToRedFraction` parameter is applied uniformly across all LAs and all GWLs. In reality, escalation probability likely increases with warming (hotter events are more likely to tip into red), and varies by region. The model holds this fraction constant as a scenario assumption.

**`costPerFamilyPerDay` values the caregiver's time, not childcare spend.** It is the Green Book opportunity cost of a lost working day at the ONS median full-time wage, applied to the supervision-weighted family count. It does not vary by region or income band, and does not distinguish paid replacement care from unpaid lost productivity — both are treated as an equivalent loss of economic output.

**Recent-climate baseline understates current warming.** UK-CRI's own methodology states the observed 1981–2010 period was 0.61°C warmer than the 1850–1900 pre-industrial average. Because the climate has continued to warm since 2010, the true present-day warming level is higher than this baseline figure — the "Recent climate" card in the dashboard carries an explicit footnote to this effect.

## 10. Event footprint (regions)

A single red heat-health alert is **not** national — UKHSA/Met Office alerts are issued for whole **regions**. Every local authority is tagged with one of the 9 English regions (`region` field in `localAuthorities.json`), derived in `build_data.mjs` from an ONS Local-Authority-District→Region lookup (`scripts/geo/lad_rgn_lookup.json`), with manual fallbacks for the reorganised unitaries whose CRI district codes predate the lookup.

The dashboard's shared control bar lets the user choose **which regions are under the alert** (the event footprint). Selecting a region adds all its LAs to the active set; the LA-level "refine authorities" selector still allows finer adjustment. The default footprint is **South East + London** (52 LAs, ~2.5m pupils) — the common serious-heat footprint, and close to the July 2022 red-alert extent. "All England" remains available as the extreme (and rare) upper bound.

Both tabs operate on the selected footprint:
- **Single event** — impact of one red alert across the selected regions (no longer implicitly national).
- **Climate outlook** — decade/career impact for the selected regions at a given GWL.

This addresses the main limitation noted previously: that a single-event calculation across all 153 LAs assumed a simultaneous nationwide red alert. The `eventsPerAuth` interpretation in §5 (average per authority for a single heatwave) is now applied within a realistic regional footprint rather than the whole country.
